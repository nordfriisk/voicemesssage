# VoiceMessage MCP Server

A Model Context Protocol (MCP) server that sends voice messages via Twilio, using local Kokoro TTS for text-to-speech and Cloudflare R2 for audio hosting.

## 🎯 Purpose

Send pre-recorded voice messages to phone numbers by:
1. **Local TTS Generation** - Converts text to WAV using Kokoro Onnx model (CPU-only, no GPU needed)
2. **R2 Storage** - Uploads audio to Cloudflare R2
3. **Twilio Call** - Places outbound call that plays the audio from R2

## 🚀 Quick Start

### Prerequisites

- Python 3.11+
- Twilio account (trial accounts supported)
- Cloudflare R2 account with public bucket

### 1. Install

```bash
# Clone or download the project
cd VoiceMessage

# Install with pip
pip install -r requirements.txt
```

### 2. Configure

Create a `.env` file and set the required environment variables:

```bash
# Twilio credentials (from https://console.twilio.com)
TWILIO_ACCOUNT_SID=your_account_sid_here
TWILIO_AUTH_TOKEN=your_auth_token_here
TWILIO_FROM_NUMBER=+14155551234

# Cloudflare R2 credentials
R2_ACCOUNT_ID=your_r2_account_id
R2_ACCESS_KEY_ID=your_r2_access_key
R2_SECRET_ACCESS_KEY=your_r2_secret_key
R2_BUCKET_NAME=voice-message-audio
R2_PUBLIC_URL=https://pub-xxxxxxxxxxxx.r2.dev

# TTS voice (default: bf_emma - British female)
DEFAULT_VOICE=bf_emma

# Available voices:
# af_heart, af_bella, af_nicole (American female)
# am_adam, am_michael (American male)
# bf_emma, bf_isabella (British female)
# bm_george, bm_lewis (British male)
```

### 3. Setup R2 Bucket

1. Cloudflare Dashboard → R2 → **Create bucket**
2. ✅ Enable **Public access**
3. Create R2 API token with **Object Read & Write**
4. Optional: Add lifecycle rule to delete objects after **1 hour**

### 4. Run MCP Server

```bash
# Start the server
python -m src.server
```

### 5. Use

From Claude, ask:
> "Send a voice message to +41987654321, saying 'Hello there!'"

```python
# Or call the MCP tool
send_voice_message(
    phone_number="+41987654321",
    message="Hello there!",
    voice="bf_emma"  # Optional voice parameter
)
```

## 🔧 Architecture

1. **TTS**: Local Kokoro Onnx model generates WAV file (always English, no language detection)
2. **R2 Upload**: boto3 uploads WAV to public bucket
3. **Twilio**: Outbound call with `<Play>` twiml pointing at R2 URL

**No web server required** - Twilio fetches audio directly from R2.

### MCP Tool

#### send_voice_message

Sends a voice message to a phone number.

**Parameters:**
- `phone_number` (str): E.164 format, e.g. `+41987654321`
- `message` (str): Text to speak (will be spoken in English)
- `voice` (str): Voice name (optional, defaults to `DEFAULT_VOICE`)

**Returns:**
- `dict`: Call SID and status

**Example:**
```python
send_voice_message(
    phone_number="+41987654321",
    message="Hello there"
)
```

## 🐛 Troubleshooting

### Trial Account Limits

Trial accounts can only call **verified** numbers:
→ Twilio Console → Phone Numbers → **Verified Caller IDs** → Add number

### R2 Public Access

Ensure bucket has **Public access** enabled in R2 dashboard.

### Twilio API Errors

Check Twilio Console for any restrictions or verified number requirements.

### Kokoro Model

Model files are located in `models/` directory. First use downloads on the first run.

### Adding New Voices

1. Download new `voices.bin` files from kokoro-onnx
2. Add to `models/` directory
3. Update voice names in TTS config

### Cleanup

- Local WAV files in `audio/` are automatically deleted before each generation
- R2 bucket files can be automatically purged with lifecycle rule

### Logging

All logs are written to the `logs/` directory with daily rotation:
- `tts_YYYY-MM-DD.log` - TTS generation logs
- `server_YYYY-MM-DD.log` - MCP server logs
- `r2_storage_YYYY-MM-DD.log` - R2 upload logs
- `twilio_client_YYYY-MM-DD.log` - Twilio call logs
- `test_YYYY-MM-DD.log` - Test logs
- `status_call_YYYY-MM-DD.log` - Status call logs

Log format: `%(asctime)s - %(name)s - %(levelname)s - %(message)s`
